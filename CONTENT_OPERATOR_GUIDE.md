# AI-FORGE 内容运营操作手册

版本：V1  
日期：2026-04-15  
适用对象：内容运营、资料整理、后台录入人员

## 1. 这份手册是做什么的

这份手册只回答一件事：

- 当你手里有一条内容，应该怎么在后台录进去

不需要理解数据库，也不需要理解底层表。

你只需要进入统一上传台 `content_intake`，按这份手册填写即可。

## 2. 先记住一条原则

每一条资料，先想清楚它属于哪一种：

| 你手里有什么 | 你应该怎么录 |
|---|---|
| 一篇文章或一个视频链接 | 填原始链接，再补至少一种可消费内容 |
| 一个自有视频文件 | 填原始链接，再上传视频文件 |
| 一个 PPT / PDF | 填原始链接，再上传 PPT 文件 |
| 一段音频 | 填原始链接，再上传音频文件或音频外链 |
| 只有整理好的中文摘要 | 填原始链接，再把摘要正文写进去 |

一句话：

- `source_url` 是原始来源
- 摘要、音频、PPT、视频这些，是会员真正会看到的内容

## 3. 后台里最常填的字段

### 基础信息

- `资料标题`：这条资料在前台显示的名字
- `简短摘要`：一句或一小段说明，告诉用户这是什么
- `主主题`：这条内容属于哪个主题
- `所属合集`：可选；只有需要归到专题里时再填
- `封面图`：可选

### 原始来源

- `来源类型`：article / video / podcast / paper / website
- `来源平台`：例如 YouTube、OpenAI Docs、Anthropic、arXiv
- `原始链接`：原始内容的链接
- `原始来源标题`：可不填
- `作者`：可不填
- `原始语言`：不确定可以先不动
- `原始发布时间`：不清楚可以先不填

### 加工内容

下面这些内容至少要有一种：

- `摘要正文`
- `摘要附件`
- `音频文件`
- `音频外链`
- `PPT 文件`
- `PPT 外链`
- `视频文件`
- `视频外链`

### 发布设置

- `会员层级`：谁可以看
- `发布方式`：
  - 保存草稿
  - 直接发布
- `发布时间`：直接发布但不填时，系统会自动补当前时间
- `内容类型`：按实际内容选

## 4. 六个常见操作场景

### 场景 A：新增一个 YouTube 视频

适用情况：

- 你想收录一个 YouTube 视频
- 会员可能看原视频，也可能主要看你整理后的版本

最少要填：

- `资料标题`
- `简短摘要`
- `主主题`
- `来源类型 = video`
- `来源平台 = YouTube`
- `原始链接 = YouTube 视频链接`
- `视频外链 = YouTube 视频链接`，或上传你自己的视频文件
- `会员层级`
- `发布方式`

建议：

- 如果中国境内用户需要稳定访问，不要只放 YouTube 链接
- 最稳的是再补一个自有视频文件、摘要、音频或 PPT

### 场景 B：上传一个 PPT 或 PDF

适用情况：

- 你已经整理好一份 PPT、PDF 或演示材料

最少要填：

- `资料标题`
- `简短摘要`
- `主主题`
- `来源类型`
- `来源平台`
- `原始链接`
- `PPT 文件`
- `会员层级`
- `发布方式`

如果是外部文档地址，也可以填：

- `PPT 外链`

### 场景 C：上传一段音频

适用情况：

- 你已经有自有音频
- 或已有一个可以访问的音频外链

最少要填：

- `资料标题`
- `简短摘要`
- `主主题`
- `来源类型`
- `来源平台`
- `原始链接`
- `音频文件` 或 `音频外链`
- `会员层级`
- `发布方式`

### 场景 D：只放原始链接，再补一个中文摘要

适用情况：

- 原始内容你不打算搬运
- 只想给会员一份整理过的摘要

最少要填：

- `资料标题`
- `简短摘要`
- `主主题`
- `来源类型`
- `来源平台`
- `原始链接`
- `摘要正文`
- `会员层级`
- `发布方式`

这是最轻的一种录入方式。

### 场景 E：先存成草稿

适用情况：

- 内容还没整理完
- 文件还没上传齐
- 你只想先把这条记录占住

怎么做：

- `发布方式` 选 `保存草稿`
- 其它能填多少先填多少
- 保存

结果：

- 系统会把这条内容按草稿处理
- 前台不会把它当成已发布内容

### 场景 F：直接发布

适用情况：

- 内容已经齐了
- 你希望资料包可以直接出现在前台

怎么做：

- `发布方式` 选 `直接发布`
- `发布时间` 可以自己填
- 如果不填，系统会自动补当前时间

结果：

- 当前时间或过去时间：会按已发布处理
- 未来时间：会按定时发布处理

## 5. 推荐录入顺序

每次新增内容，按这个顺序最稳：

1. 先填 `资料标题`
2. 再填 `简短摘要`
3. 再选 `主主题`
4. 再填 `来源类型`、`来源平台`、`原始链接`
5. 再补至少一种加工内容
6. 再选 `会员层级`
7. 最后决定是 `保存草稿` 还是 `直接发布`

## 6. 什么情况会失败

最常见的失败原因：

- 没填 `资料标题`
- 没填 `简短摘要`
- 没选 `主主题`
- 没填 `来源类型`
- 没填 `来源平台`
- 没填 `原始链接`
- 一种加工内容都没填

如果失败，后台会把原因写进：

- `失败原因`

你只需要按提示补内容，然后重新生成或重新检查。

## 7. 失败后怎么处理

### 情况 1：提示缺字段

处理方式：

- 直接把缺的字段补上
- 保存后重新执行

### 情况 2：同一个来源好像重复了

处理方式：

- 先检查 `原始链接` 是否填对
- 系统会尽量自动识别重复来源
- 如果还是看起来重复，先不要继续新增第二条，交给管理员排查

### 情况 3：文件传不上去

处理方式：

- 先确认文件本身能正常打开
- 再尝试改用外链
- 如果仍不行，先保存草稿，不要强行直接发布

### 情况 4：发布后前台没看到

先检查：

- `发布方式` 是否选成了草稿
- `发布时间` 是否填成了未来时间
- `会员层级` 是否限制了可见范围

## 8. 日常使用建议

- 不确定时，先保存草稿
- 原始链接一定要填准确
- 同一条内容不要反复新建多条记录
- 只要能用摘要解决，就不必每次都上传大文件
- 面向中国境内用户时，不要把 YouTube 当成唯一入口

## 9. 你不需要关心的东西

这些是系统内部处理的，不需要内容运营去管：

- `packages`
- `sources`
- `processed_assets`
- `package_sources`
- `package_topics`
- `package_collections`
- 排序号
- 语言字段回写
- 生成后的资料包 ID

你只需要把统一上传台这一条记录填好。

## 10. 最后记住一句话

如果你不知道该怎么填，就用这个最小思路：

- 一条原始链接
- 一段简短摘要
- 至少一种会员可消费内容
- 最后选择草稿或发布
