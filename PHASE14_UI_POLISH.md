# FargeSpace Think Tank Phase 14 UI Polish

版本：V1
日期：2026-04-10
状态：Planned

## 1. 目标

这一阶段只做一件事：

- 在不重做产品结构的前提下，把前台视觉精修成更安静、更像会员智库的样子

## 2. 当前视觉方向

保留方向：

1. 极简
2. 安静
3. 会员感
4. 像私人阅读桌，不像资讯门户

不改的东西：

1. 不重做后端数据结构
2. 不重做页面信息架构
3. 不因为接口变多而重做产品逻辑

## 3. 当前主要问题

1. 首页第一屏信息权重过于平均，没有唯一主角
2. 全站的边框、胶囊、浅色块太多，显得碎
3. 颜色语言不够收敛，品牌气质还没固定
4. 登录与阅读页还没有形成同一套会员站体验

## 4. 模块划分

### 模块 A：视觉系统

范围：

1. 颜色
2. 字体层级
3. 间距
4. 边框
5. 阴影
6. 按钮与标签

目标：

- 先统一视觉语法，再改页面

### 模块 B：共享壳

范围：

1. 顶部品牌区
2. 导航
3. 登录状态区
4. 页脚

目标：

- 让全站先有同一个会员站气质

### 模块 C：首页

范围：

1. 首屏主推荐
2. 最新内容
3. 主题入口
4. 合集入口
5. 搜索入口

目标：

- 把首页从“模块堆叠”改成“主推荐带路”

### 模块 D：会员入口页

范围：

1. 登录页
2. 忘记密码页
3. 重置密码页

目标：

- 让会员入口更完整，也更符合当前人工重置阶段

### 模块 E：内容阅读页

范围：

1. 内容详情页
2. 资产区
3. 来源区
4. 主题与合集区

目标：

- 强化阅读体验，降低视觉噪音

### 模块 F：浏览页

范围：

1. 主题页
2. 合集页
3. 搜索页

目标：

- 保持功能完整，同时统一视觉节奏

## 5. 原子任务

### T1401 冻结视觉基线

输出：

1. 视觉方向一句话
2. 颜色主副体系
3. 字体层级
4. 间距与边框规则

### T1402 收敛全局设计 token

输出：

1. `globals.css` 里的颜色变量重写
2. 间距、圆角、阴影统一
3. 按钮与标签样式统一

依赖：T1401

### T1403 精修共享壳

输出：

1. 顶部品牌区改版
2. 状态区降噪
3. 页脚信息重排

依赖：T1402

### T1404 精修首页首屏

输出：

1. 首屏改成单一主推荐叙事
2. 主标题、说明、入口按钮重排
3. 第一屏不再像仪表盘

依赖：T1403

### T1405 精修首页次级区

输出：

1. 最新内容样式重排
2. 主题入口样式重排
3. 合集入口样式重排
4. 搜索入口位置和样式重排

依赖：T1404

### T1406 精修会员入口页

输出：

1. 登录页视觉精修
2. 忘记密码页改为人工重置提示
3. 重置密码页与登录页风格统一

依赖：T1403

### T1407 精修内容阅读页

输出：

1. 详情页标题区重排
2. 资产区与来源区降噪
3. 主题/合集区统一成阅读页附属区

依赖：T1402

### T1408 精修浏览页

输出：

1. 主题页、合集页、搜索页风格统一
2. 筛选区不再拥挤
3. 列表页和详情页形成同一语言

依赖：T1405、T1407

### T1409 做桌面端与移动端视觉回归

输出：

1. 桌面端检查记录
2. 移动端检查记录
3. 需要修的视觉问题清单

依赖：T1403-T1408

## 6. 推荐顺序

1. 先做 `T1401-T1403`
2. 再做 `T1404-T1406`
3. 然后做 `T1407-T1408`
4. 最后做 `T1409`

## 7. 完成标准

满足以下条件，才算这一阶段完成：

1. 首页第一屏有明确主次
2. 全站颜色和边框语言统一
3. 登录页和阅读页属于同一套站点气质
4. 浏览页不再显得密和碎
5. 桌面端和移动端都通过回归检查
