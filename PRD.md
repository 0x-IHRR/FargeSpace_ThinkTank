## 1. 产品概述

FargeSpace Think Tank 是一个面向会员的 AI 资料平台。

平台核心不是课程，不是论坛，也不是普通文档库。平台的核心价值是：

- 人工筛选高质量海外 AI 资料
- 将原始资料加工为适合会员消费的形式
- 统一分类、整理、发布
- 让会员在一个地方高效浏览可信内容

V1 的主目标不是追求复杂功能，而是建立一套稳定、清晰、可扩展的内容后台与会员浏览体验。

## 2. 目标与非目标

### 2.1 产品目标

V1 目标：

1. 建立稳定的内容管理后台
2. 支持原始来源与加工内容的统一管理
3. 支持会员按主题、格式、合集浏览内容
4. 支持编辑从创建到发布的完整流程
5. 支持后续 UI/UX 重做而不影响底层内容结构

### 2.2 非目标

V1 明确不做：

- 用户发帖
- 社区讨论
- 学习进度
- 课程体系
- 自动抓取和全自动内容处理
- 推荐算法
- 收藏功能
- 多会员层级

## 3. 目标用户

### 3.1 会员用户

会员用户的主要诉求：

- 不想自己到处找 AI 资料
- 希望有人替他筛选高质量内容
- 希望快速看到整理后的版本
- 希望可以按主题和格式回头查找

### 3.2 编辑用户

编辑用户的主要诉求：

- 能录入原始资料
- 能上传和整理加工后的内容
- 能将多个内容打包成一个面向会员的内容包
- 能控制草稿、审核、排期、发布

### 3.3 管理员

管理员的主要诉求：

- 管理用户和角色
- 管理分类和合集
- 管理发布规则和可见性
- 管理系统配置与内容规范

## 4. 核心产品定义

平台的核心发布对象是 `内容包（Package）`。

会员不直接浏览原始来源，而是浏览整理好的内容包。

一个内容包至少包含：

- 1 个原始来源
- 1 个可供会员直接消费的加工结果

一个内容包可以包含：

- 原始文章或视频链接
- 摘要文章
- 音频版本
- 幻灯片/PDF
- 编辑后视频

平台的最小使用单位是“内容包”，不是“原始资料条目”。

## 5. V1 锁定决策

| 项目 | V1 决策 |
|---|---|
| 会员层级 | 单一会员层级 |
| 后台角色 | `admin`、`editor`、`member` |
| 发布方式 | 编辑直接在平台内创建和发布 |
| 必选内容形式 | 每个内容包必须有 1 个文字摘要/简报 |
| 可选内容形式 | 音频、幻灯片、视频 |
| 原始来源展示 | 对会员展示来源链接和来源信息，但不作为主内容 |
| 音频入口 | 音频支持存在，但不是首页主入口 |
| 收藏功能 | 不进入 V1 |
| 支持的媒体类型 | 文字、链接、音频、PDF/幻灯片、视频嵌入 |

## 6. 信息架构

### 6.1 前台信息架构

V1 前台包含以下页面：

1. 首页
2. 主题页
3. 合集页
4. 内容包详情页
5. 搜索/筛选结果页
6. 登录页

### 6.2 后台信息架构

V1 后台包含以下模块：

1. 原始来源管理
2. 加工资产管理
3. 内容包管理
4. 主题与标签管理
5. 合集管理
6. 发布流程管理
7. 用户与权限管理

## 7. 页面需求

### 7.1 首页

页面目标：

- 给会员一个清晰的主入口
- 展示最新精选内容
- 展示主题入口和合集入口
- 让会员快速进入一条整理好的内容线索

页面模块：

- 本期/本周重点内容包
- 最新上线内容包列表
- 主题入口
- 合集入口
- 基础筛选入口

页面要求：

- 首页必须有一个唯一主视觉焦点
- 不能做成满屏同权重卡片墙
- 次级信息不能压过主内容包

### 7.2 主题页

页面目标：

- 让会员按主题浏览内容包
- 强化内容归档能力

页面模块：

- 主题标题与说明
- 该主题下的内容包列表
- 筛选条件

### 7.3 合集页

页面目标：

- 展示按期次或策展逻辑组织的内容
- 支持“本周精选”“特别专题”等组织方式

页面模块：

- 合集标题与说明
- 合集内内容包列表
- 排序信息

### 7.4 内容包详情页

页面目标：

- 成为会员实际消费内容的核心页面

页面模块：

- 标题
- 摘要
- 主题与合集信息
- 原始来源信息
- 加工资产区
- 相关推荐入口

展示规则：

- 先展示加工后的内容
- 再展示原始来源信息
- 原始来源只能作为辅助说明，不可成为主位

### 7.5 搜索/筛选结果页

页面目标：

- 帮助会员按条件找到需要的内容包

筛选条件：

- 主题
- 格式
- 来源类型
- 发布时间
- 内容包类型

### 7.6 登录页

页面目标：

- 提供会员登录入口
- 提供后台用户登录入口

## 8. 后台模块需求

### 8.1 原始来源管理

必须支持：

- 新建原始来源
- 编辑原始来源
- 录入来源链接
- 录入作者、平台、语言、发布时间
- 标记来源类型

来源类型：

- article
- video
- podcast
- paper
- website

### 8.2 加工资产管理

必须支持：

- 录入摘要/简报
- 上传音频
- 上传 PDF / 幻灯片
- 配置视频嵌入
- 关联到指定内容包

资产类型：

- brief
- audio
- slides
- video

### 8.3 内容包管理

必须支持：

- 新建内容包
- 绑定一个或多个原始来源
- 绑定一个或多个加工资产
- 配置主主题
- 配置合集归属
- 配置发布时间
- 配置发布状态

业务规则：

- 内容包必须至少有 1 个原始来源
- 内容包必须至少有 1 个加工资产
- 原始来源不能单独直接面向会员发布

### 8.4 主题与标签管理

V1 至少支持：

- 主题
- 难度
- 用途
- 信号等级
- 内容包类型
- 发布周期

V1 受控值：

| 字段 | 可选值 |
|---|---|
| difficulty | beginner / intermediate / advanced |
| use case | awareness / strategy / tooling / workflow / research |
| signal level | high_signal / reference / archive |
| package type | recap / deep dive / watchlist / toolkit / interview |
| publication cycle | weekly / monthly / special |

### 8.5 合集管理

必须支持：

- 创建合集
- 设置合集说明
- 配置排序
- 绑定内容包

### 8.6 发布流程管理

发布状态：

- draft
- review
- approved
- scheduled
- published
- archived

V1 要求：

- 每个内容包必须有一个明确状态
- 发布窗口在内容包级别生效

### 8.7 用户与权限管理

角色：

- `admin`
- `editor`
- `member`

V1 要求：

- 角色权限可配置
- 会员仅能看到已发布内容
- 编辑可处理内容但不可越权做系统级设置
- 管理员拥有全部权限

## 9. 权限矩阵

| 操作 | admin | editor | member |
|---|---|---|---|
| 查看后台 | 是 | 是 | 否 |
| 管理用户 | 是 | 否 | 否 |
| 管理分类 | 是 | 是 | 否 |
| 创建原始来源 | 是 | 是 | 否 |
| 创建加工资产 | 是 | 是 | 否 |
| 创建内容包 | 是 | 是 | 否 |
| 发布内容包 | 是 | 是 | 否 |
| 查看 draft | 是 | 是 | 否 |
| 查看 scheduled | 是 | 是 | 否 |
| 查看 published | 是 | 是 | 是 |
| 浏览前台 | 是 | 是 | 是 |

## 10. 关键业务规则

1. 会员访问对象始终是内容包，不是原始来源
2. 每个内容包必须至少有一个原始来源
3. 每个内容包必须至少有一个加工资产
4. V1 中加工资产按内容包独立管理，不做跨包共享
5. 原始来源可展示给会员，但仅作为辅助信息
6. 未发布内容不可被会员访问
7. 内容结构优先于前台表现，页面由内容结构生成，而不是反过来

## 11. 用户流程

### 11.1 会员流程

1. 登录平台
2. 进入首页
3. 通过首页主入口或主题页进入内容包
4. 阅读摘要或查看加工资产
5. 查看来源信息
6. 继续进入相关主题或合集

### 11.2 编辑流程

1. 新建原始来源
2. 创建或上传加工资产
3. 新建内容包
4. 绑定来源与加工资产
5. 配置主题、合集、发布时间
6. 提交审核或直接发布

### 11.3 管理员流程

1. 维护分类体系
2. 管理编辑与会员账号
3. 管理发布规则
4. 检查内容质量与系统稳定性

## 12. V1 验收标准

### 12.1 内容管理验收

- 编辑可独立创建原始来源
- 编辑可独立创建加工资产
- 编辑可独立创建内容包
- 编辑可完成从草稿到发布的流程

### 12.2 会员体验验收

- 会员能在首页看到最新内容包
- 会员能按主题浏览内容包
- 会员能按筛选条件找到内容包
- 会员打开详情页后，能先看到整理后的内容，再看到来源说明

### 12.3 权限验收

- member 无法访问后台
- member 无法访问 draft 和 scheduled 内容
- editor 可以管理内容但不能管理用户
- admin 拥有全部权限

### 12.4 结构稳定性验收

- 用 20 条真实内容测试内容结构
- 测试过程中不能依赖临时字段和特例规则
- 若 20 条内容不能自然归类，必须先修正文档和模型，不能直接写代码补洞

- 后台：Directus

- 权限：Directus 原生角色权限

说明：

- 这是当前建议，不是不可变更的死规定
- 但在没有更强理由前，默认以这套组合进入实施拆分

## 16. 下一步建议

推荐顺序：

1. 写字段级数据结构表
2. 用 20 条真实内容跑试填
3. 再拆开发 TODO

- 最后拆 TODO，任务才不会空泛
